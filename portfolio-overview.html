Personal website

HTML5, CSS, JavaScript, Responsive design

To showcase my work, I both designed and developed this portfolio from scratch.

Design

I studied a number of online portfolios and styles to gather ideas, then built mockup in Sketch. I settled on a strong header theme with a responsive home page grid that would display six key pieces of work, and a blue color palette for backgrounds, sections and links.

Once I had a theme designed, I quickly built a layout for page content in each project. This involved setting breakpoints for a main container class, rem values and padding for recurring elements like paragraphs and image blocks. At this point, I also set up responsive typography styles in my main stylesheet.

Color palette and final logo icon design

Early mockups of home page and content pages to get a feel for the overall theme of the site

Development

I started with the home page so I could set up the header and footer styles that would appear on each page together with the project grid. I built the first draft just like the mockup, with a full-width grid of responsive sections that would house project images and an interactive tooltip on hover.

I tested a hover-based tooltip, but decided against it for mobile considerations.

I employed flexbox to style the main menus and footer links, using the breakpoints I established along with my typography to set up a display across all device widths. Developing the theme first allowed me to rapidly build each project page once it was done.

Refinement

The project grid was responsive and functional, but even varying background colors didn’t give each project tile the impact I wanted. So I went back to the drawing board.

I designed other layouts and hover states that didn't make the cut, but helped envision the final product

I redesigned a grid with gutters that would separate each project tile and give it its own space. I did some math and set margins at percentages to achieve a responsive gutter that didn’t require additional classes for rows or individual items.

Gutters and a border radius gave each project tile more impact and space of its own

Version control

I checked out different branches to test versions of the home page to see which performed best. Even with a static site, it was good pracitce to have branches that let me save versions of my code and return to them later if needed.

Optimization

Once I built a full draft of the website, I revisited my code to reduce complexity and add semantics for future reference. I added semantic HTML, partitioned my stylesheet to keep it organized, and optimized all of my page images for maximum performance.

In-development

I am currently developing an alternate home page that displays design work alongside my development work. I have several projects in branding, vector design and print layout that I want to showcase in a separate grid so users can see my full skillset.
